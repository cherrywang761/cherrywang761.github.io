1. 数组和链表的区别是什么，数组和链表在内存里是否是连续的，为什么数组可以用索引直接取数？

数组和链表是两种基本的数据类型（都属于线性表），但是在内存存储上的表现并不一样，都有各自的特点

数组与生俱来的具有线性的特质，而链表需要一个有维度的线把它的值串起来。

  • 内存存储区域

数组在内存中存储是一块连续的区域，而链表可以存放在内存中的任何位置，不要求连续。

  • 预留空间

数组是需要预留空间的，当你创建一个数组，在使用前需要申请占用内存的大小，这样有可能会造成内存浪费。

链表它是不需要预留空间的，每个数据都存放着指向下一个数据的内存地址，可以一直扩容，只要存了地址。

  • 功能

数组查找容易，增删不易：数组查找容易的原因就是因为有索引可以直接取数（连续），但是增删的话，后面的值都要移动（内存地址会发生变换）

链表增删容易，查找不易：它增删，新建一个对象，然后指向它就好了。查找要从头遍历因为没有索引（不连续）

  • 扩展性

数组不易扩展，每次扩展都要重新定义数组，链表易扩展。

2. int和Integer的区别？int++和Integer++的区别？

  1. int是基本数据类型，Integer是int对应的包装类

  2. Integer变量需要实例化后才能使用，而int变量不需要

  3. Integer实际是对象的引用，当new一个Integer是，实际上是生成一个指针指向此对象，而int则是直接用来存储数据的

  4. Integer的默认值是null，而int的默认值是0

Integer对象分为new生成和非new生成

  1. new生成的两个Integer对象是不同的，因为指向了不同的内存空间

  2. 非new生成的两个Integer对象有个新特性

原因是：java在编译Integer i = 100时，翻译成Integer i = Integer.valueOf(100)

而javaAPI中对Integer类型的ValueOf的定义如下，对于-128到127之间的数，会进行缓存，Integer i = 127时，会将127这个Integer对象进行缓存，下次再写Integer j = 127时，就会直接从缓存中取，就不会new了。

  1. 非new生成的Integer变量和new Integer()生成的变量比较时，结果为false

    ①当变量值在-128~127之间时，非new生成的Integer变量指向的是java常量池中的对象，而new Integer()生成的变量指向堆中新建的对象，两者在内存中的地址不同；

    ②当变量值不在这区间内，那么非new生成的Integer会按照new Integer(i)处理，两者在内存中的地址还是不同

3. hashmap底层

（get方法比较两key是怎么比较的）

JDK8以前，hashmap是由数组+链表组成的数组是HashMap的主体，链表则是主要为了解决哈希冲突而存在的（“拉链法”解决冲突）。

JDK8后，在解决哈希冲突时有了较大的变化，当链表长度大于阈值（默认为 8）时，将链表转化为红黑树，以减少搜索时间。

  • hashmap中一些重要的属性
  • loadFactor: 加载因子是控制数组存放数据的疏密程度，loadFactor越趋近于1，那么数组中存放的数据(entry)也就越多，也就越密，也就是会让链表的长度增加；loadFactor越小，也就是趋近于0，数组中存放的数据(entry)也就越少，也就越稀疏。
  • threshold：threshold = capacity * loadFactor；当Size>=threshold时，需要考虑扩容

Tips:链表变成红黑树结构的条件有两个：1.链表长度大于8 2.当前散列表的长度已经达到64；不然它只会发生散列表扩容（resize）

equals和hashcode

hashcode()方法是将数据的内存地址以int类型输出， equals（）方法判断两个对象是否是同一个对象（String类型不要求是同一个对象）

我们往map中插入一个object对象，首先调用hashCode()方法计算object的哈希码，通过哈希码可以直接定位object在哈希表中的位置(一般是哈希码对哈希表大小取余)。如果该位置没有对象，可以直接将object插入该位置；如果该位置有对象(可能有多个，通过链表实现)，则调用equals()方法比较这些对象与object是否相等，如果相等，则不需要保存object；如果不相等，则将该对象加入到链表中。

简单来说，我们是先判断hashcode相不相同，如果相同，则hashcode就无法判断对象是否相等了，就要祭出大杀器equals方法进行判断了

所以equals相等，hashcode一定相等

hashcode相等，equals不一定相等

对象在equals中的信息没有改变，则hashcode不变

关于hash算法的问题

Q: 什么是一个好的哈希算法？

A：散列程度大，分布均匀

Q：hash值是hashcode()得到的吗？

A:不是。hash值是key.hashcode二次加工得到的；是key.hashcode得到的值高16位^低16位得到的。

Q：为什么是h&(length-1)?

key的hashCode经过hash后，为了让其在table（table为hashMap的entry[]）的范围内，需要再hash一次。这里实际上采用的是除余法（hash%length)

计算机中直接求余效率不如位移运算，源码中做了优化，当length是2的n次方的时候hash%length==hash&(length-1)**，其次 **求余的话高位没有参与运算

Q：为什么hashmap的数组长度一定是2的幂次方？

A： 这样子能够散列的效果最好（要保证最多的1，1111的1的数量大于1110，散列效果好，因为任何数和0相与都是0，所以对于1110中，0001,0011,0101…这些位置都不能存放数据了）。

B： 每次扩容后，新数组和原数组之间只有一位差异，如果新数组和原数组最左边的差异位都为0，那么新索引位置就是旧索引位置，这样可以大大减少计算量。

Q: 为什么高低位异或？

A：为了优化hash算法，一般table.length-1得到的二进制数都在16位以内，这样子的话高16位就浪费了，所以就为了让高位参与运算，避免浪费。

4. List，Set，Map说一下

List,Set,Map都是继承自Collection类的

  • Collection是单列集合的根接口，主要用于存储一系列符合某种规则的元素，它有两个重要的子接口List和Set。
  • List是有序列表，其存放内容是可以重复的，实现接口：ArrayList，LinkedList，Vector
  • Set是无序的集合，其内部存放的内容不可以重复，实现接口：HashSet，TreeSet，LinkedHashSet
  • Map存的是键值对（key-value），key值不允许有重复，value内容可以相同，但对应不同的key，实现接口：HashMap，TreeMap，LinkedHashMap

ArrayList
  • 初始化

使用默认构造方法（无参构造）时，创建的是一个空数组，只有当第一次插入数据时，才会将数组容量初始化为10（懒加载）

  • 扩容机制

int newCapacity = oldCapacity + (oldCapacity >> 1); //每次扩容1.5倍

举个例子：往ArrayList中插入19个数

  1. 当add第一个元素时，容器是个空数组，我们就会将旧容器的容量初始化为10；
  1. 当add第二个元素时，minCapacity(理解为当前装的东西大小) - elementData.length（容器的容量） > 0不成立，所以不会调用grow()；
  2. 当add到第11个元素时，minCapacity - elementData.length > 0成立，就要调用grow（）方法；扩容到原来的1.5倍；扩容后都要比较新容量是否大于MAX_ARRAY_SIZE（Integer.MAX_VALUE-8），如果大于就将容器置为Integer.MAX_VALUE。（为啥是Integer.MAX_VALUE-8，因为数组作为一个对象，需要一定的内存存储对象头信息，对象头信息最大占用内存不可超过8字节。）
  3. 当add到16时，调用grow（）方法，扩容到22的大小；并与MAX_ARRAY_SIZE进行比较。。
  4. 最后add完第19个元素，整个过程就结束了。
  • 复制方式

ArrayList源码中用到的复制方法有两个：System.arraycopy，Arrays.copyOf

二者的区别：

  1. Arrays.copyOf本质上是调用了System.arraycopy方法
  2. arraycopy需要传入目标数组，将复制后的数据放到目标数组中，而copyOf是在方法内部创建一个数组，将内容复制到这个数组然后返回该数组（一个要两个数组参数，另一个只需要一个）
