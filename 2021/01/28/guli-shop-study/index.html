1. 环境搭建

使用docker进行环境的搭建

1. docker安装mysql

systemctl status docker #查看docker状态

docker pull mysql:5.7 #下载mysql5.7

docker images #查看docker下载的镜像

#创建实例并启动

docker run -p 3306:3306 –name mysql \ #将容器的3306端口映射到主机的3306端口

-v /mydata/mysql/log:/var/log/mysql \ #将配置文件夹挂载到主机

-v /mydata/mysql/data:/var/lib/mysql \ #将日志文件夹挂载到主机

-v /mydata/mysql/conf:/etc/mysql \ #将配置文件夹挂载到主机

-e MYSQL_ROOT_PASSWORD=root \ #初始化root用户的密码

-d mysql:5.7 #后台方式运行

Tips: 遇到端口占用

netstat -nap #查看端口占用情况

kill pid

Tips：遇到容器占用

docker ps -a #查看所有的container

docker rm xxx #删除容器

  • 进入mysql中

docker exec -it mysql /bin/bash #进入到mysql容器内部

  • 在挂载的文件夹中修改mysql的配置信息

vi /mydata/mysql/conf/my.conf #编辑配置文件

2. docker安装Redis

  • 下载镜像

docker pull redis

  • 先把配置文件创建出来（不然挂载的时候就变成目录了）

mkdir -p /mydata/redis/conf

touch /mydata/redis/conf/redis.conf

  • 启动Redis实例

docker run -p 6379:6379 –name redis \

-v /mydata/redis/data:/data \

-v /mydata/redis/conf/redis.conf:/etc/redis/redis.conf \

-d redis redis-server /etc/redis/redis.conf

  • 进入redis客户端

docker exec -it redis redis-cli

  • 配置redis持久化（默认是没有开启持久化的）

vi /mydata/redis/conf/redis.conf

搭建分布式系统的基本环境

SpringCloud-Alibaba

很多人可能会问，有了spring cloud这个微服务的框架，为什么又要使用spring cloud alibaba这个框架了？

最重要的原因在于spring cloud中的几乎所有的组件都使用Netflix公司的产品，然后在其基础上做了一层封装。然而Netflix的服务发现组件Eureka已经停止更新；而其他的众多组件也快停止维护。所以急需其他的一些替代产品，也就是spring cloud alibaba，目前正处于蓬勃发展的态式。

Nacos作注册中心

  • 导入依赖

因为所以项目都需要服务注册中心，来将他们的服务注册上去，所以将依赖导入到common项目中

  • 下载Nacos客户端

Tips: 启动时报错！

解决方法： 将startup.cmd中的set MODE=”cluster”修改为set MODE=”standalone”。

  • 需要在项目中配置Nacos服务器的地址

spring.cloud.nacos.discovery.server-addr: localhost:8848

  • 使用@EnableDiscoveryClient注解开启服务注册与发现功能（我没有这步也能被发现。。）

  • 要为每个注册服务取一个名字

spring.application.name: gulimall-coupon

  • 同理给所有项目都进行服务注册的配置

Feign声明式远程调用

Feign是一个声明式的HTTP客户端，它的目的就是让远程调用更加简单。

Feign提供了HTTP请求的模板，通过编写简单的接口和插入注解，就可以定义好HTTP请求的参数、格式、地址等信息。

Feign整合了Ribbon（负载均衡）和Hystrix（服务熔断），可以让我们不再需要显示地使用这两个组件。

  • 远程调用的步骤

    • 引入依赖

    当初搭建项目骨架的时候已经引入了open feign的依赖了

    • 编写一个接口，告诉SpringCloud这个接口需要调用远程调用，声明接口的每一个方法都是调用哪个远程服务的哪个请求

    如果我们调用的是该远程接口下的membercoupons()方法，那么就会先去找远程服务gulimall-coupon所在的位置，然后再去调用/coupon/coupon/member/list路径下的方法。

    • 开启远程调用的功能@EnableFeignClients

API网关

网关作为流量的入口，常用功能包括路由转发、鉴权、限流控制等。

springcloud gateway作为SpringCloud推出的第二代网关框架，取代了Zuul网关。

客户端发请求给服务端。中间有网关。先交给映射器，如果能处理就交给handler处理，而后要经过一系列filter，最后路由到指定的服务。

  • Route: 发一个请求给网关，网关要将请求路由到指定的服务。路由有id，目的地uri，断言的集合，匹配了断言就能到达指定位置，
  • Predicate断言: 就是java里的断言函数，匹配请求里的任何信息，包括请求头等。只要断言条件判断成功了，才给你路由到指定的地方。
  • Filter: 请求和响应都可以通过过滤器进行修改。

前端知识快速入门

ES6

ECMAScript6.0简称ES6，是JavaScript语言的下一代版本。

它是浏览器脚本语言的规范，而我们熟知的，如JavaScript则是规范的具体实现

  • let

let类似于var，是用来声明变量的。

不同于var，var声明的变量往往会越域，而let声明的变量有严格的局部作用域

var可以声明多次，而let只能声明一次

var会变量提升，而let不存在变量提升

  • const

声明常量，一旦声明，不允许改变

  • 解构表达式

#数组解构

let arr = [1,2,3];

let [a,b,c] = arr; #这样子能够快速将数组中的内容赋值到指定的变量上

#对象解构

const person = {name: “jack”, “age:21”, “language: [‘java’,’js’,’css’]”}

const {name,age,language} = person; #这样子能够快速将对象的属性快速取出来

  • 字符串插入变量和表达式

let info = (反引号)我是${abc},今年${age}岁了(反引号) #字符串要用反引号包起来哦

  • 函数参数的默认值

如果一个函数接收两个值，但是只传一个值，那么第二个值的默认值也是一个值

  • 不定参数

function fun(…values){console.log(values.length)}

fun(1,2) #输出2

fun(1,2,3,4,5) #输出5

  • 箭头函数

var print = obj => console.log(obj); #表示传一个obj参数并打印出来

var sum2 = (a,b) => a + b; #表示传入两个参数返回相加的值

  • 对象函数属性简写
  • map函数

map()：接收一个函数，将原数组中的所有元素用这个函数处理后放入新数组返回

  • reduce函数

reduce()为数组中的每一个元素依次执行回调函数，不包括数组中被删除或从未被复制的元素

  • Promise方法

Promise方法可以封装异步操作，它可以避免非常长的嵌套循环

Vue

不知道咋总结，反正实操就完事了

  • MVVM思想

M：即Model，模型，包括数据和一些基本操作

V： 即View，视图，页面渲染结果

VM：即View-Model，模型与视图间的双向操作

  • v-bind

插值表达式只能用在标签体里面，v-bind的作用就是给标签的属性绑定值

  • v-model

v-model一般用于表单项，用于双向绑定的过程

  • v-on

v-on一般是用来绑定事件的，也可以用@来绑定

  • v-for

v-for一般是用来遍历循环的，遍历的时候都加上：key来区分不同的数据，提高Vue渲染效率

v-for=”item in items” :key=”item.id”

  • v-if

v-if,顾名思义，条件判断，当得到结果为true是，所在的元素才会被渲染

v-show，当得到结果为true是，所在的元素才会被显示

v-if是连语句都没有渲染进去，而v-show是通过样式将元素隐藏起来

  • 计算属性

某些结果是基于之前数据实时计算出来的，我们可以利用计算属性来完成

  • 侦听器

watch可以让我们监控一个值的变化，从而做出相应的反应

  • 过滤器

过滤器常用来处理文本格式化的操作，过滤器可以用在两个地方：双花括号插值和v-bind表达式

钩子函数

每个Vue实例在被创建时都要经过一系列的初始化过程：创建实例，装载模板，渲染模板等。

Vue为生命周期中的每个状态都设置了钩子函数（监听函数）

每当Vue实例出不同的生命周期是，对应的函数就会被触发调用。
