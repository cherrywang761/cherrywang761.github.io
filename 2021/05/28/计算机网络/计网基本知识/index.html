计网基础知识概括

发表于 2021-05-28 更新于 2021-06-15 分类于 基础知识 ， 计算机网络 阅读次数：
本文字数： 5.3k 阅读时长 ≈ 5 分钟

计算机网络体系结构

  • 应用层 ：为特定应用程序提供数据传输服务，例如 HTTP、DNS 等协议。数据单位为报文。
  • 表示层 ：数据压缩、加密以及数据描述，这使得应用程序不必关心在各台主机中数据内部格式不同的问题。
  • 会话层 ：建立及管理会话。
  • 传输层 ：为进程提供通用数据传输服务。由于应用层协议很多，定义通用的传输层协议就可以支持不断增多的应用层协议。运输层包括两种协议：传输控制协议 TCP，提供面向连接、可靠的数据传输服务，数据单位为报文段；用户数据报协议 UDP，提供无连接、尽最大努力的数据传输服务，数据单位为用户数据报。TCP 主要提供完整性服务，UDP 主要提供及时性服务。
  • 网络层 ：为主机提供数据传输服务。而传输层协议是为主机中的进程提供数据传输服务。网络层把传输层传递下来的报文段或者用户数据报封装成分组。
  • 数据链路层 ：网络层针对的还是主机之间的数据传输服务，而主机之间可以有很多链路，链路层协议就是为同一链路的主机提供数据传输服务。数据链路层把网络层传下来的分组封装成帧。
  • 物理层 ：考虑如何传输比特流，物理层的作用是尽可能屏蔽传输媒体和通信手段的差异，使数据链路层感觉不到这些差异。

Q：路由器需要的协议？

A：路由器只有下面三层协议，因为路由器位于网络核心中，不需要为进程或者应用程序提供服务，因此也就不需要传输层和应用层。

物理层（PASS）

链路层

基本三操作：封装成帧，透明传输，差错检测

  • 封装成帧：网络层传下来的分组会添加首部和尾部封装成帧
  • 透明传输：需要对首部和尾部字段进行定界操作，通过转义字符进行判定，用户是察觉不到转义字符的存在的。
  • 差错检测：广泛使用循环冗余检验（CRC）来检测比特差错

常见协议

CSMA/CD协议

CSMA/CD表示载波监听多点接入/碰撞检测

  • 多点接入：主机以多点方式连接到主机上
  • 载波监听：主机需要不停的监听信道
  • 碰撞检测：如果监听到信道已有其他主机正在发送数据，就表示发生碰撞，就要一个碰撞的退避策略。

截断二进制指数退避算法：从离散的整数集合 {0, 1, .., (2k-1)} 中随机取出一个数，记作 r，然后取 r 倍的争用期作为重传等待时间。

PPP协议

PPP协议是用户计算机和ISP进行通信时所使用的协议

MAC地址

MAC地址是链路层地址，长度为6字节（48位），用于唯一标识网卡

有多少网卡就有多少MAC地址。

网络层

网络层可以说是整个互联网的核心，网络层向上只提供简单灵活的、无连接的、经最大努力交互的数据包服务。

使用IP协议，可以把异构的物理网络连接起来，使得在网络层看起来好像是一个统一的网络。

与 IP 协议配套使用的还有三个协议：

  • 地址解析协议 ARP（Address Resolution Protocol）
  • 网际控制报文协议 ICMP（Internet Control Message Protocol）
  • 网际组管理协议 IGMP（Internet Group Management Protocol）

IP分类

由两部分组成：网络号+主机号

CIDR记法：采用在 IP 地址后面加上网络前缀长度的方法，例如 128.14.35.7/20 表示前 20 位为网络前缀。

地址解析协议 ARP

网络层实现主机之间的数据传输，而链路层实现具体每段链路之间的数据传输。因此在通信过程中，IP 数据报的源地址和目的地址始终不变，而 MAC 地址随着链路的改变而改变。

ARP协议就通过IP地址得到MAC地址

ARP内部有个高速缓存记录局域网上的各主机和路由器的 IP 地址到 MAC 地址的映射表。

如果A知道B的IP，但是ARP中没有记录B的MAC映射，那么它会广播通知并获得MAC值写入ARP高速缓存中。

网际控制报文ICMP

ICMP是为了更有效的转发IP数据报和提高交付成功的机会。它被封装在IP数据报中。

ICMP报文分为差错报文和询问报文

常用操作：PING，他就是ICMP的一个重要应用，来测试两台主机之间的连通性

ping原理就是通过向目的主机发送ICMP Echo请求报文

目的主机收到之后会发送Echo回答报文。

传输层

网络中真正通信的并不是主机而是主机中的进程。

传输层提供了进程间的逻辑通信。

传输层最重要的两个协议：UDP和TCP

UDP

用户数据报协议，无连接，是为了保证及时性传输的，没有拥塞控制

面向报文（对应用程序传下来的报文不合并也不拆分，只添加UDP首部）

TCP

传输控制协议TCP，面向连接，是为了保证可靠性传输的，有流量控制，拥塞控制，面向字节流（应用层传下来的报文当做字节流，把字节流组织成大小不等的数据块），TCP只能一对一

三握四挥

三次握手

Client想和Server建立连接，于是主动发送TCP握手请求

  • C向S发送连接诶请求报文，SYN = 1, ACK = 0, 选择一个初始序号seq= x
  • S接收到请求报文，如果同意建立连接，向A发送连接确认报文，SYN = 1, ACK = 1, 确认号ack = x+1, 同时也选择一个初始序号 seq= y;
  • C收到S发送的确认报文后，还要最终核实一下，向B发送确认, ACK = 1, 确认号ack = y+1，序号为x+1
  • S收到确认后，建立连接

为什么要三次握手？

保证C，S的收发都没有问题，这样能防止失效的连接请求到服务器，让服务器错误打开连接，消耗资源

第三次握手取消掉行不行？

不行啊

  1. 如果取消掉的话，S端不知道C收和S发正不正常

  2. 倘若有网络问题，S端第二次握手返回的确认报文很慢，C端等了一个超时重传时间还没到，它就要重新请求连接。

    结果重新请求完，这个确认报文到了，如果没有第三次握手，那么服务器就会对滞留的连接请求进行确认（但实际上它应该是过期了）

四次挥手

在建立连接后ACK = 1

Client想与Server结束连接

  • A发送连接释放报文， FIN = 1,选择初始序号seq = u
  • B收到释放报文后进行确定，ACK=1,ack = u+1, 选择一个初始序号seq=v，这时候TCP属于半关闭状态（CLOSE WAIT)，B能向A发送数据但是A不能向B发送数据
  • 等B发送完数据后，不需要维持这个连接了，就发送释放报文给A，FIN=1,ACK=1,ack=u+1,选择一个初始序号seq = w
  • A收到后，进入TIME-WAIT状态，等待2MSL(最大报文存活时间)后释放连接，ACK=1, ack = w+1, seq = u+1
  • B收到A的确认后释放连接

为什么要四次挥手？

  1. CLOSE_WAIT过程是为了让服务端发送完所有数据。
  2. TIME_WAIT过程的2MSL时间：（为什么要等2MSL呢？）
  1. 确保最后一个A发向B的确认报文能够到达。如果B没收到A发送的确认报文，那么会重新发送连接释放请求报文，等2MSL这段时间就是为了处理这种情况
  2. 等一段时间也是为了让本次连接产生的所有报文都从网络中消失，使得下次新连接出现旧连接报文

拥塞控制

拥塞控制为了降低整个网络的拥塞程度。

因为如果网络出现拥塞，分组将会丢失。所以发送方会再次重传数据，从而导致网络的拥塞程度更高，这样的恶性循环必须避免，因此当出现拥塞时，应当控制发送方的速率。

TCP主要通过四个算法来进行拥塞控制：

慢开始、拥塞避免、快重传、快恢复

cwnd：拥塞窗口，是一个状态变量，但实际发送多少数据取决于发送方窗口

ssthresh：慢开始门限

  • 慢开始

最初令cwnd为1，当cwnd < ssthresh时，每次cwnd加倍

当cwnd >= ssthresh时，进入拥塞避免

  • 拥塞避免

当进入拥塞避免：每个轮次cwnd加1

如果超时，令ssthresh = cwnd / 2，并重新执行慢开始

  • 快重传

发送方传输数据给接收方，先发送M1数据，接收方接收到并返回接收M1数据的确认报文，接着发送M2，假设发送M3数据时，数据丢失了，当发送方发送M4数据时，接收方发现M3没有确认，就会要求重复确认M2，发送M5，再次要求重复确认M2，发送M6，第三次重复确认M2，因为三次重复确认请求就会触发快重传机制，这时候发送方的第一要务就是把M3给发出来..

  • 快恢复

应用层

DNS域名系统

DNS是一个分布式数据库，提供了主机名和IP地址之间相互转换的服务。

分布式的意思就是每个站点只保留它自己的那部分数据

DNS域名有层级结构，自上而下为：根域名、顶级域名、二级域名

DNS虽然可以使用UDP和TCP进行传输，但是大部分情况下都是使用UDP进行传输的，使用端口号为53。

以下两种情况DNS会使用TCP进行传输：

  • 因为UDP最大只支持512字节的数据，所以当返回的响应超过512字节时候，会进行TCP进行传输
  • 区域传送（区域传送指的是主域名服务器向辅助域名服务器传送变化的那部分数据）

文件传输协议（FTP）

A: 如果有两台不同的主机，使用不同的操作系统，不同的字符集，如何进行文件传输呢？

Q: 可以采用FTP的传输方式，也可以使用Telnet协议进行不同操作系统之间的文件传输。但是FTP与Telnet的区别在于FTP使用2个TCP连接传输文件，且不需要强制两端都使用同一个标准。

为什么要使用两个连接呢？

要是用一条连接的话，端口既要处理数据连接数据，又要处理控制连接，很容易出错，而且效率很低。区分控制连接和数据连接后，数据连接的速度要快一些。

2个连接
  • 控制连接

控制连接使用端口21，主要用于在两个主机间传输控制信息，如用户标识、口令相关数据，相关命令从客户端传送给服务端，同时将服务端的应答传送给客户端

  • 数据连接

用于传输一个文件数据，使用端口20

2种模式

根据数据连接是否是让服务端主动建立的，分为主动模式和被动模式

而两者建立连接的过程，就是两次TCP请求的构建，只不过两种模式的数据连接构建并不大相同。。

  • 主动模式

服务端主动建立数据连接，服务器端的端口号为20，客户端端口号随机，但必须大于1024（因为0~1023是熟知端口号）

主动模式的利弊

主动模式对FTP服务器的管理有利，因为FTP服务器只需要开启21端口的“准入”和20端口的“准出”即可。

但这种模式对客户端的管理不利，因为FTP服务器20端口连接客户端的数据端口时，有可能被客户端的防火墙拦截掉。

  • 被动模式

服务端不是主动建立数据连接，而是由客户端主动建立数据，且客户端的端口号也由自己决定，服务端端口号随机

被动模式的利弊

被动模式对FTP客户端的管理有利，因为客户端的命令端口和数据端口都是“准出”，windows防火墙对于“准出”一般是不拦截的，所以客户端不需要任何多余的配置就可以连接FTP服务器了。

但对服务器端的管理不利。因为客户端数据端口连到FTP服务器的数据端口P时，很有可能被服务器端的防火墙阻塞掉。

动态主机配置协议（DHCP）

DHCP协议提供了即插即用的联网方式，用户不需要再手动的配置IP地址等信息

相关信息包括：IP地址，子网掩码，网关IP地址等

远程登录协议（Telnet协议）

该协议用于一台主机登录到远程主机上，且能够获得远程主机的输出。

一次TCP

可以适应两台主题不同的操作系统和编码集（同FTP）

电子邮件协议

一个邮件系统有三部分组成：用户代理、邮件服务器、邮件协议。

邮件协议包括发送协议和读取协议

发送协议通常使用SMTP协议，读取协议使用POP3和IMAP

访问Web页面过程（面试）

  1. DHCP配置主机信息

  • 当主机开始没有IP地址以及相关信息，就需要使用DHCP进行获取
  • 主机生成DHCP请求报文，将该报文放入具有目的端口67和源端口68的UDP报文段中
  • 该UDP报文段需要放到IP数据报中进行传输（广播IP目的地址255.255.255.255）和源IP地址（0.0.0.0）
  • IP数据报需要放置在MAC帧中，将广播到与交换机连接的所有设备上
  • 连接在交换机的DHCP服务器收到广播帧后，会不断向上分解得到IP数据报，UDP数据报、DHCP请求报文，根据DHCP请求报文会得到DHCP ACK报文。。。在封装（UDP握手）
  • 最后通过DHCP报文，配置IP地址，子网掩码和DNS服务器的IP地址

  1. ARP解析MAC地址

因为DHCP过程只能知道网关路由器的IP地址，为了获取网关路由器的MAC地址，需要使用ARP协议

  • 主机通过浏览器生成TCP套接字，该套接字向HTTP服务器发送HTTP请求。为了生成套接字，需要知道网站域名
  • 主机生成ARP查询报文，然后封装报文成帧，通过交换机转发该帧给所有连接设备
  • 路由器收到该帧后，向上分解得到ARP报文，进行匹配，成功就返回一个ARP回答报文给主机，其中包括它的MAC地址。

  1. DNS解析域名

  • 知道网关路由器的MAC地址后，就可以进行DNS的解析过程了。

  1. HTTP请求页面

  • 三次握手建立连接后，浏览器生成HTTP GET报文，获得HTTP响应报文，抽取出存放在报文主体的Web页面内容，进行渲染显示出Web页面
