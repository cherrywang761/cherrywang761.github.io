线程通信方式

发表于 2021-05-19 分类于 面试 阅读次数：
本文字数： 5.4k 阅读时长 ≈ 5 分钟

如何让两个线程按顺序执行？

想要让A先执行完后，再让B执行

普通的new两个线程，无法控制先后顺序

这样子处理显然无法让A,B有顺序的执行。

如果希望B在A执行完后才执行，可以使用thread.join()方法

在线程B中添加让步方法join()

B print: 1
A print: 1
B print: 2
A print: 2
A print: 3
B print: 3

线程通信方式其一：join()

如何让线程以有序方式相交？

如果现在我们希望B在A打印完1之后立即开始打印1，2，3，然后A继续打印2，3呢

这就需要更加细粒度的锁进行控制了，可以使用object.lock()和object.notify()的优势

练习代码如下

A 1
B 1
B 2
B 3
A 2
A 3

线程通信方式其二：object的lock()和notify()方法

在A,B,C都完成同步执行后，执行D

前面说过如果有先后顺序，可以使用thread.join()方法，但是如果想让多个线程同步执行，最后在执行D，如何实现？

Tips：普通使用thread.join()会让ABC顺序执行，起不到同步作用。

三个线程同步运行，当三个线程全部完成运行后，D才开始运行

这里我们可以使用CountdownLatch计数器功能

  1. 创建一个定时器，设置一个初始值
  2. countDownLatch.await()进入等待状态，直到计时器变为0
  3. countDownLatch.countDown()在其他线程调用该方法，使计数器减一
  4. 当计数器为0，countDownLatch.await()等待线程中方法将立即退出并执行方法

练习代码如下

D is waiting for other three workers
C is working
A is working
B is working
Cfinished
Bfinished
Afinished
All done, D starts working

因此，CountDownLatch适用于一个线程需要等待多个线程的情况。

线程通信方式其三：CountDownLatch计数器

三个跑步者准备跑步

三个跑步者，在各自准备好了开跑。

这里三个线程A,B,C需要分别进行准备，然后在三个人都准备好后开始同时运行。如何实现？

CountDownLatch用于计数下降，完成计数后的实施者是主线程，具有不可重复性。所以只有一个线程的一个await()会被响应。

CyclicBarrier的下一步动作实施者还是“其他线程”本身，具有往复多次实施动作的特点。我们这里可以使用它来实现。

  1. 首先创建一个公共对象CyclicBarrier，并设置同时等待的线程数
  2. 这些线程会开始同时进行准备，准备好后要等待其他人完成准备工作，所以要调用cyclicBarrier.await()方法来等待其他人
  3. 当所有人都调用cyclicBarrier.await()方法时，意味着都准备好了，那么这些线程将开始同时执行。

实现代码如下

Bis preparing for time:626
Cis preparing for time:812
Ais preparing for time:914
Bis prepared, waiting for others
Cis prepared, waiting for others
Ais prepared, waiting for others
AStarts running
BStarts running
CStarts running

线程通信方式其四：CyclicBarrier 信号量

子线程将结果返回到主线程

在开发中，我们会遇到需要fork子进程执行一些耗时任务，执行完后将结果返回给主线程，那么如何具体实现呢?

需要返回结果，显然需要使用Callable接口来实现

Callable通常会和FutureTask结合使用

例如我们希望子线程计算1到100的总和，计算完毕后将结果返回给主线程，具体的代码如下：

Before futureTask.get()
Task starts
Task finished and return result
Result:5050
After futureTask.get()

主线程调用futureTask.get()方法时，会阻塞主线程。

子线程Callable开始在内部执行并返回操作结果，通过get()方法获取结果后，主线程结束阻塞，继续运行

Tips: 在这里，我们可以了解到FutureTask和Callable可以直接在主线程中获得子线程的结果，但是它们会阻塞主线程。当然，如果您不想阻塞主线程，可以考虑使用ExecutorService将FutureTask线程放入线程池来管理执行。
